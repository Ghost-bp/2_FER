## 一、学习内容总览

### 1.1 ResNet 论文（He et al., 2015）

**核心问题**：网络越深，训练误差反而上升——不是过拟合，是梯度消失导致深层网络优化困难。

**核心方案**：残差连接 `H(x) = F(x) + x`。不直接学目标映射，而是学"残差" F(x)。反向传播时加法不衰减梯度，让深层也能收到有效信号。

**核心收获**：
- BasicBlock（18/34 层）：两个 3×3 卷积 + 恒等连接，`expansion=1`
- Bottleneck（50/101/152 层）：1×1 压缩 → 3×3 提取 → 1×1 扩展，`expansion=4`，减少深层参数量
- `[2,2,2,2]` 四个 stage，空间尺寸逐步减半（56→28→14→7），通道数逐步翻倍（64→128→256→512）
- 个人手写了 ResNet 完整代码，与项目 `models.py` 结构完全对应

### 1.2 CLIP 论文（Radford et al., 2021）

**核心思想**：打破传统"固定类别标签"训练范式，用**自然语言作为监督信号**。4 亿图文对训练，图像和文本各一个编码器，通过对比学习让配对的图文向量靠近、非配对远离。

**双塔架构**：
- 图像编码器（ViT-B-32）：图片 → 512 维向量
- 文本编码器（Transformer）：文本 → 512 维向量
- 训练目标：最大化 N 对正样本的余弦相似度，最小化 N²-N 对负样本

**在本项目中的角色**：只用图像编码器，CLIP 作为冻结的"视觉语义知识库"，通过 512 维特征提供通用视觉语义（嘴巴形状、眼睛状态等），ResNet 通过门控决定激活哪些维度。

### 1.3 迁移学习

**核心逻辑**：大模型在大数据上学到的底层特征（边缘、纹理、形状）是通用的 → 加载预训练权重 → 适配小数据任务。

**本项目的双重迁移学习**：
- ResNet-18（MSCeleb 人脸识别预训练）→ `load_state_dict(strict=False)` → 参与训练
- CLIP ViT-B-32（4 亿图文对预训练）→ 完全冻结 → `torch.no_grad()` 下调用

### 1.4 交叉验证

**核心逻辑**：小数据集下随机划分 train/test 不稳定，需要多次划分取平均。

**KFold vs LOSO**：
- KFold：随机分 K 份，每份轮流当验证集 — 同一人的图可能同时出现在训练和验证
- LOSO（Leave-One-Subject-Out）：按人划分，每人轮流当验证集 — 确保验证集的人训练集完全没见过
- 本项目 12 个受试者用 LOSO，测的是"对陌生人的泛化能力"，更严格也更真实

### 1.5 损失函数

- **交叉熵（CrossEntropyLoss）**：分类任务标准损失，衡量预测分布和真实分布的距离
- 本项目三损失组合：`1.0×CE + 1.5×Masked_CE + 5.0×Diversity`，分别对应主分类、掩码鲁棒性、特征多样性

### 1.6 CNN 基础

- 卷积层（Conv2d）：提取局部特征
- 批归一化（BatchNorm2d）：稳定训练、加速收敛
- ReLU：非线性激活
- 最大池化（MaxPool2d）：降采样、增大感受野
- 全连接层（Linear）：特征 → 类别分数的映射

- **supervisor()** — 作用在融合特征上的监督正则：loss_1（掩码 CE）防止捷径依赖，loss_2（多样性）鼓励均匀贡献

- **LOSO 划分** — `[(验证人, 其余11人列表, [验证人]) for 每个受试者]`

### 3.4 数据分析

#### 3.4.1 实验 3 准确率下降的原因

实验 1/2 高达 71%+ 的准确率是**虚假的**——YOLO 检测的是"人"而非"脸"，裁剪区域覆盖整身（平均 1541×1194 px，占原图 95.8%），模型实际上看到了**衣服、背景、身体姿态**等非表情信息，学到了错误的关联。

实验 3 只投喂**真实人脸区域**（约 115×115 px），模型被迫从表情本身做判断，准确率 62.66% 是**更诚实的评估**。

#### 3.4.2 受试者表情缺失与准确率的关系

```
表情完整受试者 (3, 5, 7):  平均准确率 = 78.59%
表情缺失受试者 (4, 11, 12): 平均准确率 = 45.82%
差值: 32.77 个百分点
```

受试者 11 缺失"开心"类别（验证集 100 张全是负样本），准确率仅 42%——模型对训练集中不存在的表情类别无法泛化。

#### 3.4.3 过拟合现象

- 训练准确率普遍攀升至 **89-92%**，验证准确率停滞在 40-83%
- 训练/验证准确率差距达 **20-30 个百分点**
- CE Loss 持续下降，而 MC Loss（监督分支）几乎不变（~1.60, ~0.99）
- 说明：1106 张图 12 人的小数据集是过拟合的根本原因

## 四、项目学习总结

### 4.1 对 CAFE 架构的理解

CAFE 解决的核心问题是：**如何在小数据集上做好表情识别？**

它的答案是"借力"——借助 CLIP 在 4 亿图文对上训练好的视觉语义，然后用 ResNet 学习一个**门控信号**来决定激活 CLIP 的哪些维度：

```
输入人脸 → { CLIP(ViT-B-32, 冻结) → 512维语义特征
           { ResNet-18(可训练) → sigmoid → 512维门控信号
                           ↓
                 融合 = 语义特征 × 门控信号
                           ↓
                      FC(512→6) 分类
```

这不是简单的特征拼接，而是一种**软选择机制**：sigmoid 输出 0~1 之间的值，CLIP 不同维度的语义被不同程度地激活或抑制。

### 4.2 核心创新点

1. **门控融合（Gating Fusion）**：`image_features × sigmoid(ResNet(x))` 不是注意力机制，不是拼接，而是逐元素乘法。ResNet 决定"听 CLIP 的哪些维度"，CLIP 提供"听到了什么"。

2. **监督分支（Supervisor）**：三损失设计防止模型走捷径。随机 mask 14% 维度 + 多样性正则 = 强制所有特征维度均匀参与。

### 4.3 数据集与评估协议

- **KMU-FED**: 1,106 张图，12 人，6 类表情，无中性样本
- **核心问题**: 8/12 受试者至少缺失一种表情，表情分布不均衡（DI 仅 120 张）
- **LOSO 评估**: 每人轮流当验证集，衡量"对新人泛化能力"
