## 📖 论文背景

本项目基于论文 **《Generalizable Facial Expression Recognition》** (Zhang et al., BUPT, 2024)。

**核心问题：** 在一个数据集上训练的 FER 模型，换到另一个数据集上准确率崩塌（域差异 → 过拟合）。

**CAFE 解决方案（3 组件）：**

```
输入人脸 → CLIP(冻结) ──→ 512维 ──┐
    │                                ├→ ×(逐元素乘) → FC(6类)
    └→ ResNet-18 → sigmoid ──→ 门控 ──┘
                                    ↑
                     sigmoid 输出 0~1，选择 CLIP 的哪些维度被激活

1. Mask (sigmoid门控): CLIP × sigmoid(ResNet) — 核心创新，贡献 +8.8%
2. Separation (通道分离): 512维分成6组对应6个表情 — 替代FC层
3. Diverse (多样性损失): 强制各组均匀贡献，防止偷懒 — 贡献 +0.4%
```

## 第一阶段：ResNet 残差网络 ✅ 已完成

> 📝 Day 1: [LEARNING_LOG/day1_resnet.md](LEARNING_LOG/day1_resnet.md)

### 核心知识点

**残差连接 `x += identity`：** 加法在反向传播时不衰减梯度，解决了深层网络退化问题。

**BasicBlock（ResNet-18/34）：**
```python
# 两个 3×3 卷积，expansion=1，通道数不变
Conv3×3 → BN → ReLU → Conv3×3 → BN → +skip → ReLU
```

**Bottleneck（ResNet-50/101/152）：**
```python
# 1×1 压缩 → 3×3 提取 → 1×1 扩展×4，节省深层计算量
Conv1×1 → BN → ReLU → Conv3×3 → BN → ReLU → Conv1×1 → BN → +skip → ReLU
```

**ResNet 版本对比：**

| 模型 | Block | n_blocks | 参数 |
|------|-------|----------|:----:|
| ResNet-18 | BasicBlock | [2,2,2,2] | 11.7M |
| ResNet-34 | BasicBlock | [3,4,6,3] | 21.8M |
| ResNet-50 | Bottleneck | [3,4,6,3] | 25.6M |
| ResNet-101 | Bottleneck | [3,4,23,3] | 44.5M |

## 第三阶段：ResNet 家族 + EfficientNet 🆕

### ResNet-18 vs ResNet-34 vs ResNet-50

**关键差异：**
- **ResNet-34**: 比 18 多 16 个卷积层，每层更多 BasicBlock。更深 → 更强的特征提取，但参数量翻倍
- **ResNet-50**: 用 Bottleneck 块，1×1 卷积压缩通道 → 3×3 提取 → 1×1 扩展。参数比 34 少但更深

**为什么 Phase 2 要试这些？**
Phase 1 最佳模型 (ResNet-18, 71.94%) 只用 11M 参数。更大的 backbone 可能提取更好的特征，但要防止过拟合（小数据集的风险）。

### EfficientNet-B0
- 通过 **NAS（神经网络架构搜索）** 自动设计的网络
- 核心思想：**复合缩放** — 深度、宽度、分辨率同时按比例增加
- 仅 ~5M 参数，比 ResNet-18 小一半
- 在 ImageNet 上达到 ResNet-50 级别的精度

## 第四阶段：CLIP 与门控融合 🆕

### CLIP (Contrastive Language-Image Pre-training)

**核心思想：** 用自然语言代替固定标签。4 亿图文对训练，图像编码器 + 文本编码器各出 512 维向量，配对的拉近、不配对的推远。

**双塔架构：**
```
图像塔 (ViT-B-32):  图片 → 12层 Transformer → 512维
文本塔 (Transformer): "a happy face" → 12层 Transformer → 512维
训练目标: 最大化 N 对正样本的余弦相似度，最小化 N²-N 对负样本
```

### Sigmoid 门控机制

```python
fused = image_features * torch.sigmoid(resnet_features)
# sigmoid → 输出 0~1，实现"软选择"
# 每个维度独立控制：接近 1 = 激活，接近 0 = 抑制
```

**为什么不用 concat 或 attention？**
- Concat: 简单但无法区分哪些 CLIP 维度有用
- Attention: 需要额外参数，在小数据集上容易过拟合
- 门控: 参数量最小，ResNet 决定"听 CLIP 哪些维度"

### Phase 1 核心发现：CLIP 是噪音！

```
CLIP 层数: 12 → 9 → 6 → 3 → 0 (纯ResNet)
准确率:  60.9 → 66.0 → 63.5 → 62.5 → 71.9%
```

**CLIP 越多 → 越差。** 因为 CLIP 在 LAION-400M 上学的是物体/场景语义，不是面部肌肉的细微变化。对于 FER 来说 CLIP 特征是噪音。

## 第五阶段：优化器对比 🆕

### Adam (当前基线)
```
m_t = β₁·m_{t-1} + (1-β₁)·g_t       # 动量（一阶矩）
v_t = β₂·v_{t-1} + (1-β₂)·g_t²      # 自适应学习率（二阶矩）
θ_t = θ_{t-1} - lr · m̂_t / (√v̂_t + ε)
```
- 自适应学习率，每个参数有自己的学习率
- 收敛快，但可能泛化不如 SGD

### AdamW
```
θ_t = θ_{t-1} - lr · (m̂_t / (√v̂_t + ε) + wd·θ_{t-1})
```
- 关键改进：权重衰减与自适应学习率**解耦**
- Adam 的 L2 正则等价于在损失里加，但会被自适应学习率稀释
- AdamW 直接在参数更新时减去 wd·θ，效果更好

### SGD + Momentum
```
v_t = μ·v_{t-1} + g_t
θ_t = θ_{t-1} - lr · v_t
```
- 没有自适应学习率 → 需要更仔细的 LR 调优
- 但泛化能力通常更好（更少的捷径）
- Nesterov 加速：先看"冲过去之后"的梯度

## 第六阶段：学习率调度器 🆕

### ExponentialLR（当前基线）
```python
lr_t = lr_0 × γ^t    # γ=0.9, 每轮 LR 衰减 10%
```
- 平滑指数衰减
- 简单但缺乏适应性

### CosineAnnealingLR
```python
lr_t = lr_min + 0.5×(lr_max - lr_min)×(1 + cos(π×t/T))
```
- 余弦曲线衰减 → 先快后慢
- 在周期末尾给很小的 LR → 精细调整
- 常用于现代训练方案（配合 AdamW）

### ReduceLROnPlateau
```python
if val_acc 连续 patience 轮不提升:
    lr = lr × factor    # factor=0.5 → 减半
```
- 自适应：验证指标停滞时才降 LR
- 适合训练时间不固定的场景

## 第七阶段：正则化技术 🆕

### Dropout
```python
# 训练时：随机丢弃 p% 的神经元
# 推理时：所有神经元 × (1-p)
x = nn.Dropout(p=0.5)(x)
```
- **原理**：每次训练的模型结构不同 → 隐式的模型集成
- **为什么有效**：防止神经元之间的"共适应"（co-adaptation）
- p=0.5 是最常用的（最大正则化强度）

### Label Smoothing（标签平滑）
```python
# 不用 Label Smoothing:
y_true = [0, 0, 1, 0, 0, 0]    # one-hot
# 用 Label Smoothing 0.1:
y_true = [0.017, 0.017, 0.917, 0.017, 0.017, 0.017]  # 软化
```
- **原理**：不让模型对正确标签过于自信
- **效果**：提升泛化，减少过拟合
- 特别适合小数据集和类别不平衡的场景

### Weight Decay（L2 正则）
```python
# 在优化器中设置 weight_decay
optimizer = AdamW(params, lr=0.0002, weight_decay=1e-4)
```
- 惩罚大权重 → 让模型倾向于更简单的解
- 太大 → 欠拟合；太小 → 过拟合
- AdamW 中 weight_decay 与学习率解耦，效果更好

### 当前基线

### ColorJitter（颜色增强）
```python
ColorJitter(brightness=0.2, contrast=0.2, saturation=0.2, hue=0.1)
```
- 改变亮度、对比度、饱和度、色调
- 提高对光照变化的鲁棒性

### 几何增强
```python
RandomRotation(degrees=15)          # 旋转 ±15°
RandomAffine(translate=(0.1,0.1))   # 平移 10%
```
- 模拟头部倾斜/偏转
- FER 场景下特别重要（真实场景人脸不总是正对镜头）

### MixUp
```python
x_mix = λ·x_i + (1-λ)·x_j
y_mix = λ·y_i + (1-λ)·y_j
# λ ~ Beta(α, α), α=0.2
```
- 混合两张图及其标签
- 强制模型学习线性插值 → 更平滑的决策边界
- 对小数据集特别有效

## 第九阶段：消融实验方法 🆕

### 什么是消融实验（Ablation Study）？
- 系统性地移除/改变一个组件，观察性能变化
- **控制变量法**：只改变一个因素，其他保持不变

### Phase 2 的消融策略
```
Group A: Backbone 不同 → 回答"哪个网络最好？"
Group B: LR 不同 → 回答"最优学习率是多少？"
Group C: 优化器/调度器不同 → 回答"怎么优化最好？"
Group D: 正则化不同 → 回答"怎么防过拟合？"
Group E: 增强不同 → 回答"什么增强有效？"
Group F: 组合最优 → 回答"收益能叠加吗？"
```

### 如何看实验对比
- **Mean Acc**: 10 折平均准确率 → 越高越好
- **Std**: 标准差 → 越低越好（模型更稳定）
- **时间**: 训练总时间 → 越短越好
- **参数量**: 模型大小 → 越小越好

## 第十阶段：交叉验证 🆕

### KFold (10-Fold)
```
数据随机分 10 份 → 9 份训练 + 1 份验证 → 轮换 10 次
```

### LOSO (Leave-One-Subject-Out)
```
12 个人 → 每人轮流当验证集 → 12 折
训练集: 其余 11 人的所有图片
验证集: 1 人的所有图片
```

### 为什么 LOSO 更严格？
- KFold 可能把同一人的图分到训练+验证 → **数据泄漏**
- LOSO 保证验证集的人训练集**完全没见过**
- LOSO 测的是"对新人的泛化能力"
