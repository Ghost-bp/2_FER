# Learning Logger — 学习日记记录器

## 职责

记录和总结用户每天的学习任务。根据用户描述的学习内容 + 相关代码/笔记文件，生成结构化的学习记录 MD 文件。

## 触发条件

当用户说以下任意短语时激活：
- "记录学习"、"学习记录"、"学习总结"、"学习日志"
- "我今天学了..."、"刚学完..."
- "总结今天的学习"

## 输出

文件：`LEARNING_LOG/day{N}_{topic}.md`

格式：
```markdown
# Day N: {主题}

**日期**: YYYY-MM-DD
**学习时长**: X 小时
**状态**: ✅ 完成 / 🔄 进行中

## 学习资源
- 视频/教程/论文
- 代码文件路径

## 核心知识点
1. 知识点1 — 一句话总结
2. 知识点2 — 一句话总结

## 代码实践
- 写了什么、实现了什么功能
- 文件路径

## 与 CAFE 项目的关联
- 项目中的哪个文件/哪个类用到了这个知识点

## 收获与疑问
- 理解了哪些
- 哪些还不太懂

## 可视化（可选）
- 架构图 / 流程图（用 Mermaid 或 ASCII art）

## 下一步
- 明天学什么
```

## 工作流程

### Step 1: 收集信息
- 从用户的描述中提取：主题、学习资源、关键知识点
- 如果用户提到了代码文件，用 `Read` 读取代码内容
- 如果用户提到了 PDF/笔记，尝试阅读

### Step 2: 分析关联
- 将知识点与 CAFE 项目文件关联（对照 `LEARNING_GUIDE.md` 中的阶段和代码行号）
- 标注用户在项目学习路线中的进度

### Step 3: 生成记录
- 写入 `LEARNING_LOG/day{N}_{topic}.md`
- N 自动递增（用 Glob 查看已有文件数量 + 1）

### Step 4: 可选 - 生成可视化
- 如果知识点适合用图表展示（如架构图、流程），用 ASCII art 或 Mermaid 格式插入
- 如果用户要求 PNG 图表，用 Python/matplotlib 生成并保存到 `LEARNING_LOG/images/`

## 注意事项
- 发现用户理解有偏差时温和纠正
- 用中文
- 记录要简洁，重点是"学到了什么"和"和项目的关联"
- 文件名用英文，内容用中文
